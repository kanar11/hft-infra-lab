# Measured Benchmarks

This file is generated by `scripts/run_benchmarks.sh`. Two ways to refresh it:

**Option A — CI (no local Linux needed).** Go to
[Actions → benchmarks](https://github.com/kanar11/hft-infra-lab/actions/workflows/benchmarks.yml)
and click **Run workflow**. A GitHub-hosted Ubuntu runner builds, measures, and
auto-commits the result back to `main` (~2 minutes). Output is also uploaded as
an artifact for download.

**Option B — local.** On any Linux box with g++ ≥ 9:

```bash
scripts/run_benchmarks.sh           # writes BENCHMARKS.md at repo root
```

Until either runs, the README's per-module performance figures (`oms/README.md`,
`risk/README.md`, etc.) reflect the original development hardware. The
`build-and-test` job in `.github/workflows/tests.yml` also runs every benchmark
on each push — raw numbers live in that run's job log even if this file isn't
refreshed.

## What gets measured

| Module                 | Knob                         | Reported                            |
|------------------------|------------------------------|-------------------------------------|
| Orderbook              | 1M random add/cancel/match   | wall time, ops/sec                  |
| Orderbook latency hist | 100k orders                  | min, p50, p95, p99, p99.9, max ns   |
| SPSC queue             | 10M producer→consumer        | throughput msg/sec, avg latency ns  |
| Memory / cache latency | walk N-byte buffer           | ns/access by working-set size       |
| ITCH parser            | 1M synthetic messages        | msg/sec                             |
| OMS                    | 100k submit + fill           | orders/sec, p99 latency ns          |
| Risk Manager           | 100k check_order             | checks/sec, p99 latency ns          |
| Smart Router           | 100k route_order             | routes/sec, p99 latency ns          |
| Trade Logger           | 100k log calls               | events/sec, latency percentiles ns  |
| Mean Reversion         | 100k price ticks             | decisions/sec                       |
| FIX / OUCH             | 100k parse                   | msg/sec                             |
| Market Simulator       | 50k pipeline end-to-end      | E2E throughput, P&L                 |
